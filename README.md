# 마비노기 경매장 데이터 분석 및 추천 시스템 - 프로젝트 탐정 보고서

---

## 1. 프로젝트 소개 (짧고 재밌게!)

이 프로젝트는 마치 마비노기 경매장의 **비밀 요원** 같아요! 게임 속 경매장 데이터를 쓱싹 모아서, 어떤 아이템이 핫한지, 가격은 어떻게 변하는지, 심지어 어떤 아이템을 사면 돈을 벌 수 있을지까지 족집게처럼 알려주는 똑똑한 시스템이랍니다. 복잡한 경매장 속에서 길 잃은 밀레시안들을 위해, 똑똑한 정보로 무장시켜주는 것이 이 프로젝트의 핵심 목표죠!

## 2. 핵심 기능 요약 (기능도 친근하게!)

*   **경매장 데이터 수집 및 처리: "정보 수집가와 정리의 달인"**
    *   `mabinogi_client.py`는 마비노기 경매장을 샅샅이 뒤져 아이템과 거래 내역을 싹 다 긁어모으는 **정보 수집가** 역할을 합니다.
    *   `model/` 폴더의 친구들은 수집된 정보들을 깔끔하게 정리해서 담아두는 **데이터 보관함**이죠.
    *   `processor/`에 있는 녀석들은 수집된 데이터를 필터링하고, 요약하고, 필요한 정보만 쏙쏙 뽑아내고, 심지어 정렬까지 해주는 **정리의 달인**들입니다. (예: `mabinogi_auction_item_filter.py`는 불필요한 데이터를 걸러내는 **데이터 체** 역할을 하고, `mabinogi_ohlc_extractor.py`는 가격 변동의 핵심만 뽑아내는 **가격 탐정**이죠!)

*   **데이터 분석 및 추천 서비스: "미래를 예측하는 점술가와 수익 계산기"**
    *   `bot/service/recommend_service.py`는 어떤 아이템이 지금 가장 핫하고 유용한지 알려주는 **미래를 예측하는 점술가**입니다.
    *   `bot/revenue/calculate_revenue.py`는 특정 아이템을 사고팔았을 때 얼마나 이득을 볼 수 있을지 꼼꼼하게 계산해주는 **수익 계산기** 역할을 톡톡히 해냅니다.

*   **웹 인터페이스: "친절한 정보 안내 데스크"**
    *   `templates/index.html`, `templates/chart.html`은 사용자들이 정보를 쉽고 편하게 볼 수 있도록 예쁘게 꾸며진 **정보 안내 데스크**입니다.
    *   `templates/style.css`는 이 안내 데스크를 더욱 멋지고 보기 좋게 만들어주는 **스타일리스트** 역할을 하죠.

*   **Discord 봇 통합: "내 손안의 경매장 비서"**
    *   `bot/guild_client.py`는 Discord 서버에 상주하며, 사용자가 궁금한 정보를 물어보면 척척 대답해주는 **경매장 비서**입니다.
    *   `bot/storage/`는 이 비서가 자주 쓰는 아이템 목록이나 추천 아이템 목록을 잊지 않도록 잘 보관해두는 **비서의 메모장**이라고 할 수 있겠네요.

## 3. 주요 모듈/클래스 소개

*   **`adapter/mabinogi/`**: 마비노기 경매장과 직접 소통하는 **외교관** 같은 곳입니다.
    *   `mabinogi_client.py`: 경매장 데이터를 직접 가져오는 **정보원**입니다.
    *   `model/`: 가져온 데이터들을 담는 **데이터 컨테이너**들이 모여있습니다. (`AuctionItemDto`, `AuctionHistoryDto` 등)
    *   `processor/`: 가져온 데이터를 요리조리 가공하는 **데이터 요리사**들이 살고 있습니다. (필터링, 집계, 추출, 정렬 등)

*   **`bot/`**: Discord 봇의 두뇌와 팔다리 역할을 하는 곳입니다.
    *   `guild_client.py`: Discord 서버에서 봇을 움직이는 **봇의 심장**입니다.
    *   `service/recommend_service.py`: 사용자에게 꿀팁을 제공하는 **추천 전문가**입니다.
    *   `revenue/calculate_revenue.py`: 돈 계산을 담당하는 **재무 담당관**입니다.
    *   `storage/`: 봇이 기억해야 할 중요한 정보들을 저장하는 **봇의 기억 저장소**입니다.

*   **`src/`**: 프로젝트의 핵심 로직과 웹 관련 파일들이 모여있는 **중앙 사령부**입니다.
    *   `db/mabinogi_db_connector.py`: 데이터베이스와 연결해주는 **데이터베이스 통역사**입니다.
    *   `service/`: 다양한 비즈니스 로직을 처리하는 **전문 서비스 팀**들이 있습니다.
    *   `templates/`: 웹 페이지의 뼈대와 스타일, 그리고 움직임을 담당하는 **웹 페이지 건축가**들입니다. (`index.html`, `style.css`, `chart.js` 등)

## 4. 데이터 흐름 (유쾌하게 그림 없이 설명)

이 프로젝트의 데이터는 마치 신나는 여행을 떠나는 것과 같아요!

1.  **출발! (입력)**: `mabinogi_client.py`라는 용감한 탐험가가 마비노기 경매장으로 떠나 아이템 정보와 거래 내역을 잔뜩 수집해옵니다.
2.  **정비 시간 (처리)**: 수집된 데이터는 `model/` 친구들에게 잠시 맡겨져 깔끔하게 정리되고, `processor/`의 데이터 요리사들이 불필요한 정보는 걸러내고, 필요한 정보는 맛있게 요리합니다.
3.  **분석과 예측 (처리)**: 잘 요리된 데이터는 `recommend_service.py`와 `calculate_revenue.py` 같은 똑똑한 분석가들에게 전달되어, 어떤 아이템이 돈이 될지, 가격은 어떻게 변할지 예측하고 추천 정보를 만들어냅니다.
4.  **정보 전달! (출력)**: 마지막으로, 이 모든 유용한 정보는 두 가지 방법으로 사용자에게 전달됩니다.
    *   **웹 인터페이스**: `templates/` 폴더의 웹 페이지를 통해 예쁜 차트와 함께 한눈에 보기 쉽게 보여집니다.
    *   **Discord 봇**: `guild_client.py`를 통해 Discord 채팅창에서 사용자가 물어보면 즉시 답변해줍니다.

이렇게 데이터는 수집되고, 가공되고, 분석되어 사용자에게 유용한 정보로 변신하는 멋진 여정을 떠난답니다!

## 5. 사용된 기술과 도구

이 녀석들 덕분에 프로젝트가 날아갑니다!

*   **언어**: Python (파이썬은 만능 재주꾼이죠!)
*   **웹 프레임워크**: 아마도 Flask나 FastAPI 같은 가벼운 웹 프레임워크가 사용되었을 것 같습니다. (웹 인터페이스를 보니 그런 느낌적인 느낌!)
*   **데이터베이스**: `src/db/mabinogi_db_connector.py`를 보니 데이터를 저장하고 관리하는 데이터베이스가 사용되는 것 같습니다. (어떤 종류인지는 더 봐야 알겠지만, 아마 SQLite나 PostgreSQL 같은 녀석일 거예요!)
*   **Discord 연동**: `bot/` 폴더를 보니 Discord API를 활용하는 라이브러리 (예: `discord.py`)가 사용되었을 겁니다.
*   **프론트엔드**: HTML, CSS, JavaScript (웹 페이지를 만들고 꾸미고 움직이게 하는 기본 삼총사!)
